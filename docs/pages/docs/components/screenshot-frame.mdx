import { ScreenshotFrame } from "zudoku/components";

Use **ScreenshotFrame** for marketing-style product shots and documentation hero images: a mesh
gradient backdrop, asymmetric padding for corner placement, and a floating inner surface with
shadow.

## Import

```tsx
import { ScreenshotFrame } from "zudoku/components";
```

You can also use the component directly in MDX without importing it; it is included in Zudoku’s
default MDX components as `ScreenshotFrame`.

## Props

```ts
type ScreenshotFramePosition =
  | "center"
  | "top-left"
  | "top-right"
  | "bottom-left"
  | "bottom-right"
  | "bottom"
  | "left"
  | "right";

type ScreenshotFrameTheme =
  | "primary" // uses primaryColor to build the mesh
  | "ocean"
  | "sunset"
  | "neon"
  | "aurora"
  | "forest"
  | "twilight"
  | "rose";

type ScreenshotFrameProps = {
  children: React.ReactNode;
  /** Default `"bottom-right"`. */
  position?: ScreenshotFramePosition;
  /** Default `"ocean"`. Use `"primary"` with `primaryColor` for brand-aligned gradients. */
  theme?: ScreenshotFrameTheme;
  /** Hex color when `theme` is `"primary"`. Default `#3b82f6`. */
  primaryColor?: string;
  /** Outer frame: `"4/3"` (default) or `"auto"` with a minimum height. */
  aspect?: "4/3" | "auto";
  caption?: string;
  className?: string;
};
```

### Position

**Horizontal gutter is always symmetric** (same inset on start and end). The inset scales with frame
width using a container query (`clamp` + `cqi`), so small frames keep a modest band and large frames
get a thicker one (capped at `4rem`).

On corner and edge modes, the screenshot can still **flush to the block edge** (top or bottom) so it
sits against that side of the outer radius; left and right always keep the same gutter.

| Value          | Alignment        | Flush (no gutter) |
| -------------- | ---------------- | ----------------- |
| `bottom-right` | End + bottom     | Bottom only       |
| `bottom-left`  | Start + bottom   | Bottom only       |
| `top-right`    | End + top        | Top only          |
| `top-left`     | Start + top      | Top only          |
| `bottom`       | Centered, bottom | Bottom only       |
| `left`         | Centered, start  | None              |
| `right`        | Centered, end    | None              |
| `center`       | Middle           | None              |

### Themes

| Theme      | Description                                      |
| ---------- | ------------------------------------------------ |
| `ocean`    | Cyan, deep blue, magenta, and soft light accents |
| `sunset`   | Warm oranges, pink, and purple                   |
| `neon`     | High-contrast magenta, cyan, and violet          |
| `aurora`   | Greens and teals over a cool base                |
| `forest`   | Deep greens with natural highlights              |
| `twilight` | Indigo and violet with soft mist                 |
| `rose`     | Rose, blush, and cream tones                     |
| `primary`  | Mesh derived from `primaryColor` (HSL-based)     |

See **[Theme previews](#theme-previews)** below for a visual of each preset.

### Advanced

For custom layouts you can reuse the same gradient math:

```tsx
import { meshStyleFromPrimary } from "zudoku/components";

const style = meshStyleFromPrimary("#7c3aed");
// { backgroundColor, backgroundImage } — spread onto any element
```

## MDX (no import)

```mdx
<ScreenshotFrame theme="twilight" position="top-right">
  ![API console](/images/api-console.png)
</ScreenshotFrame>
```

:::info

Children are usually an `<img />` or any React node. Images scale down with `max-width` /
`object-contain` so they stay inside the inner rounded frame.

:::
