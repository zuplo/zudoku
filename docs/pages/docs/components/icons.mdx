An icon is an [Iconify](https://iconify.design) `prefix:name` string, rendered with the `Icon`
component from `zudoku/icons`. You can mix icon sets freely. Hover any of these to see its name:

## Usage

Pass the `prefix:name` to `icon`, with an optional `size` (pixels) and `className`:

```tsx
import { Icon } from "zudoku/icons";

<Icon icon="lucide:rocket" />;
<Icon icon="ph:rocket-launch-duotone" size={32} className="text-primary" />;
```

## Lucide is built in

The [Lucide](https://lucide.dev/icons) set ships with Zudoku, so it needs no installation. It is
also the default set: a bare name with no prefix resolves to it.

```tsx
<Icon icon="rocket" /> // same as "lucide:rocket"
```

## Adding another icon set

Every other set is opt-in. Install it, then reference its icons by prefix:

<Stepper>

1. Find the set's prefix on [icon-sets.iconify.design](https://icon-sets.iconify.design) (Phosphor
   is `ph`, Material Design Icons is `mdi`, Tabler is `tabler`, …).
2. Install its `@iconify-json/<prefix>` package as a dev dependency:

   ```sh
   npm install --save-dev @iconify-json/ph
   ```

3. Reference icons from it with `prefix:name`:

   ```tsx
   <Icon icon="ph:rocket-launch-duotone" />
   ```

</Stepper>

The `@iconify-json` package is read only at build time, never shipped to the browser. Only the icons
you reference are inlined.

:::info

Lucide is the only set bundled by default. Any `prefix:name` from a set you haven't installed will
be treated as [unresolved](#unresolved-icons).

:::

## Icons in configuration

The same `prefix:name` string works wherever navigation accepts an `icon`, including the
[navigation config](/docs/configuration/navigation#icons) and document
[frontmatter](/docs/markdown/frontmatter#sidebar_icon):

```ts
{ type: "category", label: "Getting Started", icon: "lucide:book" }
```

## Unresolved icons

If an icon can't be resolved at build time (a typo, or a set that isn't installed):

- In development it is fetched from the Iconify API, and the build logs a warning naming the
  unresolved icon.
- In production it renders nothing by default. Install the relevant `@iconify-json/*` package, or
  opt into runtime fetching:

```ts
// zudoku.config.tsx
export default {
  icons: { runtimeFetch: true },
};
```

## Lucide components (deprecated)

For backwards compatibility, `zudoku/icons` still re-exports the
[lucide-react](https://lucide.dev/icons/) components:

```tsx
import { AtomIcon } from "zudoku/icons";

<AtomIcon size={42} />;
```

This is deprecated and will be removed in a future major version. Prefer the string-based `Icon`
component.
